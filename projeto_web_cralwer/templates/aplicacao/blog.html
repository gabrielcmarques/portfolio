Processo da construção do app Web Crawler. Objetivos:

- Escanear um website a procura de links dentro de hrefs

- Armazenar localmente

- Imprimir no browser

- Permitir funções para deletar ou editar manualmente o link e label

- Disponibilizar uma API em formato JSON

Para realizar cada tarefa, utilizei a linguagem que mais domino no momento, Python com o framework Django e Django Rest Framework. Também irei refazer em Javascript NodeJS, linguagem que comecei a estudar recentemente em Agosto 2022, foi um ótimo exercício.

Comecei criando uma aplicação web simples em Django com 3 arquivos HTML: um para aceitar um 'texto', 'input edit' e botão para 'submit'. O de editar com 'input', 'cancelar' e 'submit', e cancelar com 'input delete' e 'cancelar'.

Configurando o MVC do django, Models.py, Urls.py e Views.py, e o principal, Crawler.py para realizar a função de escanear o website e realizar os filtros. Nesse arquivo utilizei a biblioteca 'requests' que faz requisições GET para consulta, e 're', que realiza filtros de Regular Expression para filtrando strings de texto.

Primeiro o arquivo realiza as importações das libraries e do Modelo "Artigo", que molda automaticamente um objeto da forma que desejarmos, nesse caso para receber um Titulo e uma URL. A função 'webCrawler' vai receber a URL do website como argumento, cria duas listas vazias, uma para itens duplicados, outra para os links, realizamos a consulta GET, convertemos para UTF8 pois por padrão ele não aceita símbolos e cedilhas, filtra com Regular Expression procurando por links dentro de 'hrefs', com 'https://', contendo letras, traços, pontos e sublinhados.

O loop condicional vai filtrar cada link, checar se tem duplicados, dar juntar na lista dupe_list caso tenha. a função link.split vai separar a string do link com "." como referência, ex: www.google.com -> "www" "google" "com", "[1]" no final vai selecionar apenas o elemento "google".

Após realizar o filtro, chamamos o modelo Artigo e damos os argumentos para criar o objeto filtrado, o modelo vai fazer isso para todos os links que foram validados no processo e salvar localmente.

E finalmente, retornamos a função para a views.py, que vai chamar armazenar como contexto para o arquivo crawler.html, que foi configurado para imprimir cada elemento dentro do modelo Artigo que estiver disponível. Com Titulo, URL, botão de Editar e Deletar. Estilizei tudo com Bootstrap e CSS.
